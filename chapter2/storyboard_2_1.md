# Thomas Calculus 2.1 — 变化率与切线
## Manim 动画分镜设计文档

> **核心问题（全片灵魂）**
> 速度需要"一段时间"才能测量——那么"这一瞬间"的速度，到底是什么意思？

---

## 一、视频定位

| 项目 | 内容 |
|:--|:--|
| 目标观众 | 微积分零基础初学者 |
| 预计时长 | 9–11 分钟 |
| 场景数量 | 10 个场景 |
| 叙事结构 | **悖论 → 观察 → 直觉 → 代数确认 → 推广** |
| 情绪弧线 | 困惑 → 好奇 → 逐渐清晰 → 啊哈 → 满足 |

---

## 二、视觉设计规范

### 颜色语义（全片统一，不得混用）

| 颜色 | 色值 | 语义 | 使用场合 |
|:--|:--|:--|:--|
| 冰蓝 | `#58C4DD` | 时间 / 横轴 / 运动 | x轴、时间标注、水平变化量 Δt |
| 草绿 | `#83C167` | 距离 / 纵轴 / 函数值 | y轴、距离标注、竖直变化量 Δy |
| 亮黄 | `#FFFF00` | 结论 / 切线 / 关键点 | 切线、啊哈时刻、最终结论文字 |
| 暖橙 | `#FF9D00` | 割线 / 过渡状态 | 割线（从橙色旋转成黄色，暗示"正在趋近切线"） |
| 亮红 | `#FF4455` | 矛盾 / 无法计算 | 除以零、左右极限冲突 |
| 浅白 | `#EEEEEE` | 普通文字 / 坐标轴 | 叙述性文字、轴线 |
| 深灰 | `#1C1C2E` | 背景 | 全片统一背景色 |

### 透明度分层规则

```
主角元素（当前场景的核心）  → opacity 1.0
支撑元素（坐标轴、标注）    → opacity 0.65
背景参考（上一场景的残留）  → opacity 0.25
已完成的信息（不再是重点）  → opacity 0.4，不删除
```

### 字体规则
- 所有中文文字：等宽字体（Consolas / Menlo）
- 公式：MathTex，字号不小于 36pt（低质量渲染下可读）
- 重要结论：字号不小于 48pt，配合 `Underline` 或颜色强调

### 动画节奏原则
- 每个**核心概念**出现后，强制等待 ≥ 2.0 秒
- 场景8（割线→切线）总停顿时间 ≥ 7 秒
- 同一场景内，相邻动画之间最小间隔 0.5 秒
- 代数推导每一行单独出现，间隔 1.0–1.5 秒

## 三、场景详细设计

---

### 场景 0：开场黑屏提问（10 秒）

**情绪目标：** 制造安静感，让观众准备好"思考"

**布局：** 全黑背景，居中一行白字

**动画序列：**

1. 黑屏，0.5 秒后，中央出现一行字（`Write`，1.5s）：

   > **"你能测量一辆停着的车的速度吗？"**

2. `Wait(2.0)` — 让问题停留

3. 文字淡化至 30% 透明度，留在屏幕上

4. 下方出现标题（`FadeIn`，1.0s）：

   > **变化率与曲线的切线**

5. `Wait(1.5)` — 进入场景1

**设计意图：**
开门见山用悖论，而不是"今天我们学习……"。把标题放在问题**之后**出现，让观众先进入思考状态，再知道"原来这就是今天的主题"。

---

### 场景 1：悖论——除以零（30 秒）

**情绪目标：** 让观众真正被"卡住"，不是模糊的困惑，而是具体的数学死局

**布局：** 居中，分两个阶段

**视觉元素：**
- 左侧：一段路程示意图（水平线段，标注起点终点）
- 右侧：逐步建立的公式卡片

**动画序列：**

**第一阶段（平均速度——观众已知）：**

1. 画出水平路程线段，左端标 `t₁`，右端标 `t₂`（蓝色）— 1.0s
2. 用绿色括号标注路程差 `Δs`，蓝色括号标注时间差 `Δt` — 1.0s
3. 右侧出现公式（逐字写出）：

   ```
   平均速度 = Δs / Δt
   ```

4. 代入具体数字：`60 km / 1 h = 60 km/h` — 0.8s
5. `Wait(1.5)` — 观众点头："这我懂"

**第二阶段（瞬时速度——困难出现）：**

6. 屏幕上方出现新问题（黄色，`Write`，1.2s）：

   > **"那'此刻'的速度呢？"**

7. 时间区间开始收缩动画：`Δt` 对应的蓝色括号从右向左压缩 — 2.0s
   - 同时：右侧 `Δt` 的数值从 `1` 缩小到 `0.1`，再到 `0.01`，再到 `0`

8. 当 `Δt = 0` 时，公式变为（红色）：

   ```
   速度 = Δs / 0 = ???
   ```

9. 用红色 `Circumscribe` 圈住 `/ 0` — 1.0s
10. `Wait(3.0)` — **关键停顿，让死局悬在那里**

11. 屏幕右下角出现小字（白色，30% 透明度）：

    > *"这个问题困扰了数学家两千年。"*

12. `Wait(1.5)`

**设计意图：**
不用抽象提问，用"除以零"这个具体的数学事实来制造冲突。观众会感到"我知道这是不对的，但我不知道怎么绕过去"——这种具体的被卡住感，比"有意义吗"更能驱动接下来的学习动力。

---

### 场景 2：自由落体——先看运动，再见公式（35 秒）

**情绪目标：** 建立具体的物理直觉，让公式感觉像是"对刚才看到的东西的描述"

**布局：** 左侧三分之一为物理场景，右侧三分之二为数据区域

**视觉元素：**
- 左侧：一个小圆点（石块），从顶部向下移动，轨迹为竖直虚线
- 右侧上方：时间计数器（蓝色）
- 右侧下方：下落距离计数器（绿色）
- 最后出现：公式 `y = 16t²`

**动画序列：**

1. 左侧出现悬崖边缘轮廓（淡白色，简洁）— 0.8s
2. 石块（白色小圆点）出现在顶部 — 0.3s
3. **石块开始下落**（`MoveAlongPath`，匀加速感，总时长 2.5s）
   - **同时**：右侧时间计数器从 `0.0` 增加到 `2.0`（蓝色 `DecimalNumber`）
   - **同时**：右侧距离计数器从 `0` 增加到 `64`（绿色 `DecimalNumber`）
   - **同时**：左侧石块旁用绿色虚线标注当前下落高度

4. 石块落底，静止 — `Wait(1.5)` — 让观众感受"它确实越落越快"

5. 右侧上方出现公式（`Write`，1.5s）：

   ```
   y = 16t²
   ```

   - `t` 用蓝色标注，`y` 用绿色标注，`16` 用白色

6. 用带箭头的注释线，从公式的 `t²` 指向"下落越来越快"的轨迹 — 1.0s

7. `Wait(2.0)` — 让观众建立"这个公式就是刚才那个运动"的印象

**设计意图：**
公式必须**最后**出现，并且通过颜色和箭头与已经看到的物理现象绑定。观众不应该感到"给了一个公式，然后用它来算"，而是"我们先看到了运动，然后给它起了个名字"。

---

### 场景 3：平均速度——从两个时刻（40 秒）

**情绪目标：** 确认基础工具，同时埋下"区间依赖"的隐患

**布局：** 主视觉居中偏上，公式在右下角

**视觉元素：**
- 竖直时间轴（左侧，蓝色）
- 两个时刻：`t=0`（顶部）和 `t=2`（下方），对应石块位置
- 绿色括号标注 `Δy = 64`
- 蓝色括号标注 `Δt = 2`
- 公式卡片

**动画序列：**

1. 时间轴从场景2的石块运动中"抽象"出来（`Transform`，1.5s）
   - 石块的运动轨迹 → 竖直时间轴
   - 这一步要平滑，让观众感受到"我们还是在说同一件事"

2. 在 `t=0` 处标记点（蓝色圆点）— 0.5s
3. 在 `t=2` 处标记点（蓝色圆点）— 0.5s

4. `GrowFromCenter` 绘制绿色括号，标注 `Δy = 64`（绿色）— 1.0s
5. `GrowFromCenter` 绘制蓝色括号，标注 `Δt = 2`（蓝色）— 1.0s

6. 右下角出现公式，逐步构建（`Write` + `TransformMatchingShapes`，2.0s）：

   ```
   平均速度 = Δy / Δt = 64 / 2 = 32 ft/s
   ```

7. `Wait(2.0)`

8. **关键一步**：屏幕右上角出现一行小字（白色，`Write`，1.0s）：

   > *"结果依赖于你选的区间。"*

   这行字先以 40% 透明度出现，静静留在那里。

9. `Wait(1.5)`

**设计意图：**
第8步是关键伏笔。观众现在看到 32 ft/s，感觉这是个确定的答案，但右上角的小字轻轻暗示"这不是唯一的答案"——为下一场景的矛盾爆发做铺垫。小字的低透明度表示"这只是个提醒，不是今天的重点"，避免干扰当前场景的节奏。

---

### 场景 4：矛盾爆发——不同区间，不同结果（35 秒）

**情绪目标：** 让观众真正感受到"平均速度不够用"，主动想要一个更好的工具

**布局：** 三列并排（左 | 中 | 右）

**视觉元素：**
- 左卡片：区间 `[0, 2]` → 平均速度 `32 ft/s`
- 右卡片：区间 `[1, 2]` → 平均速度 `48 ft/s`
- 中间：一个指向 `t=2` 这个点的箭头 + 大问号

**动画序列：**

1. 左卡片从左侧滑入（`FadeIn` + `Shift`，1.0s）
2. `Wait(1.2)` — 让观众读完左卡

3. 右卡片从右侧滑入（`FadeIn` + `Shift`，1.0s）
4. `Wait(1.2)` — 让两个结果并列停留

5. 中间出现一个大箭头，指向时间轴上 `t=2` 的位置（黄色，1.0s）
6. 箭头旁边出现大问号（红色，`GrowFromCenter`，0.8s）

7. `Wait(1.5)`

8. 屏幕下方出现结论性问题（白色，大字，`Write`，1.5s）：

   > **"同一时刻，两种'速度'——哪个才是真的？"**

9. **`Wait(3.0)`** — **最长停顿之一，让矛盾自己说话**

10. 右上角原来的小字（"结果依赖于你选的区间"）此时亮度提升到 100%，用黄色 `Circumscribe` 圈住 — 1.0s

11. `Wait(1.5)` — 让观众意识到"刚才的提示，现在成真了"

**设计意图：**
场景3结尾的伏笔在这里收回。观众会有一种"哦！那行小字不是随便说说的"的感觉，这种"前后呼应"的结构会让叙事感觉更严密、更真实。

---

### 场景 5：逼近——观察数列收敛（55 秒）

**情绪目标：** "有什么东西在那里等着我们"——不是证明，是观察

**布局：** 左侧：时间轴 + 动态区间；右侧：实时更新的数值

**视觉元素：**
- 左侧：时间轴，固定点 `t₀=2`（黄色星形标记），可收缩的区间 `[2, 2+h]`（暖橙色线段）
- 右侧：两列表格（`h` | `平均速度`），逐行出现
- 区间线段颜色：从橙色（h大）渐变为黄色（h→0），暗示"越来越接近某个特殊状态"

**动画序列：**

1. 左侧建立时间轴，标记 `t₀=2`（黄色星形，`GrowFromCenter`）— 1.0s
2. 画出初始区间 `[2, 3]`（暖橙色线段，h=1）— 0.8s
3. 右侧出现表格框架（两列标题：`h` | `平均速度`）— 0.5s

4. **逐行演示**（每行流程相同，节奏从慢到快）：

   **第1行（h=1，慢速演示，让观众理解原理）：**
   - 左侧区间保持 h=1 — 停留 0.8s
   - 右侧第1行数字出现：`1` | `48` — 1.0s
   - 用细线从右侧数字连到左侧区间（0.5s），然后线淡出
   - `Wait(1.0)`

   **第2行（h=0.1，稍快）：**
   - 左侧区间收缩至 h=0.1（`ValueTracker` 动画，0.8s）
   - 区间颜色轻微偏黄
   - 右侧第2行：`0.1` | `65.6` — 0.8s
   - `Wait(0.8)`

   **第3行（h=0.01，更快）：**
   - 左侧区间收缩至 h=0.01（0.6s）
   - 区间颜色更黄
   - 右侧第3行：`0.01` | `64.16` — 0.6s
   - `Wait(0.6)`

   **第4–5行（h=0.001, 0.0001，快速出现）：**
   - `LaggedStart`，间隔递减（0.4s, 0.3s）
   - 左侧区间几乎缩为一点，颜色完全变黄，与 `t₀=2` 的标记融合
   - 右侧数字快速跳动：`64.016` → `64.0016`

5. 右侧数值列最后一行停顿 — `Wait(1.5)`

6. 在数值列右侧画一条竖虚线（黄色），然后在竖线右侧出现（`Write`，1.0s）：

   ```
   → 64
   ```

7. **`Wait(2.5)`** — 关键停顿，让"64"这个数字停在空气中

8. 屏幕下方出现（白色，`Write`，1.5s）：

   > **"我们没有'除以零'，但答案正在浮现。"**

9. `Wait(2.0)`

**设计意图：**
第4步的"逐行演示 + 细线连接"，让每一行数字都和左侧的区间收缩**物理对应**，观众不会觉得表格是从天而降的，而是看着它被计算出来的。颜色从橙到黄的渐变，是一个无声的预告："这个过程在趋向某种特殊状态。"

### 场景 6：视角切换——从物理到图像（40 秒）

**情绪目标：** 平滑过渡，让观众感受到"我们还是在说同一件事，只是换了个视角"

**布局：** 全屏，完整的坐标变换动画

**视觉元素：**
- 起点：左侧物理时间轴 + 右侧数值表（场景5的结果）
- 终点：标准二维坐标系，抛物线 `y = 16t²`，点 `P(2, 64)` 和 `Q(2+h, 16(2+h)²)`

**动画序列：**

1. 场景5的画面停留，半透明化（整体降至 50%）— 1.0s

2. 坐标轴"生长"动画（`Create`，2.0s）：
   - 左侧物理时间轴 → x 轴（蓝色）：用 `Transform` 将时间轴变形为横轴
   - 右侧数值列 → y 轴（绿色）：数值列"旋转"成纵轴
   - 这两个变换同时进行，观众看到"时间变成横轴，距离变成纵轴"

3. 场景5的内容完全淡出，只留下坐标轴 — 1.0s

4. 抛物线从左到右绘制（`Create`，蓝色曲线，2.0s）

5. 标记点 `P(2, 64)`（黄色实心圆，`GrowFromCenter`）— 0.5s
   - 旁边标注：`P(2, 64)`（白色小字）

6. 标记点 `Q(3, 144)`（暖橙色实心圆，初始 h=1）— 0.5s
   - 旁边标注：`Q(2+h, 16(2+h)²)`（白色小字）

7. `Wait(1.5)`

8. 屏幕角落出现提示（40% 透明度，白色小字）：

   > *"一个物理问题，变成了一个几何问题。"*

9. `Wait(1.5)`

**设计意图：**
坐标轴的"生长"动画要让观众看到"时间轴→x轴"和"距离值→y轴"这两个对应关系，否则坐标系会感觉是凭空出现的。这个转变应该让观众有一种"啊，所以函数图像其实是在记录运动过程"的感受。

---

### 场景 7：割线——平均变化率的几何身份（45 秒）

**情绪目标：** 建立"割线斜率 = 平均变化率"这个核心等式的几何直觉

**布局：** 全屏坐标系，右上角公式区

**视觉元素：**
- 抛物线（蓝色）
- 点 `P`（黄色）和点 `Q`（暖橙色）
- 连接P、Q的割线（暖橙色）
- 直角三角形：水平边 `Δt`（蓝色），竖直边 `Δy`（绿色）
- 右上角公式区（逐步建立）

**动画序列：**

1. 从场景6直接开始，P和Q已在屏幕上

2. 从 P 画水平虚线，从 Q 画竖直虚线，两线交于直角点 — 1.0s
   - 水平线标注 `Δt`（蓝色括号）
   - 竖直线标注 `Δy`（绿色括号）

3. `Create(割线PQ)`（暖橙色实线，从P延伸到Q，并适当延长超出P、Q）— 1.5s

4. 右上角出现公式（逐步 `Write`，每步 1.0s）：

   ```
   斜率 = 竖直变化 / 水平变化
        = Δy / Δt
        = 平均变化率
   ```

   - 写"Δy"时，坐标系中绿色竖边同步高亮（`Indicate`）
   - 写"Δt"时，蓝色横边同步高亮
   - 写"平均变化率"时，割线同步高亮

5. **`Wait(2.5)`** — 让这个等价关系沉淀

6. 屏幕上方出现总结框（黄色边框，`Create`，1.5s）：

   ```
   割线斜率 = 平均变化率
   ```

7. `Wait(2.0)`

8. **快速展示三个例子**（每个约4秒，`LaggedStart`，淡入淡出替换）：

   - **例1**：果蝇种群曲线（S型），割线标注 `≈ 8.6 只/天`
   - **例2**：股票价格曲线（锯齿状），割线标注 `平均涨幅`
   - **例3**：温度变化曲线（正弦状），割线标注 `平均升温率`

   每张图仅停留4秒，重点在于让观众感受到**这个工具无处不在**，不需要理解细节。

9. 三个例子淡出，回到抛物线 — 1.0s
10. `Wait(1.0)`

**设计意图：**
第4步的"公式写出时，图形对应元素同步高亮"是关键——让符号和图形在**同一时刻**建立联系，而不是"先看图，再看公式"或"先看公式，再想图"。

三个快速例子放在这里（原分镜中的场景12），因为这是"概念刚刚建立"的最佳时机，观众会感受到工具的力量。

---

### 场景 8：★ 割线旋转成切线——全片啊哈时刻（75 秒）

**情绪目标：** 不是理解，是**看见**——观众应该感受到"那条线就在那里等着"

**布局：** 全屏，极简，去掉一切不必要的标注

**视觉元素：**
- 抛物线（蓝色，opacity 0.7）
- 固定点 `P`（黄色，大圆点，opacity 1.0）
- 可移动点 `Q`（暖橙色，较小圆点，opacity 0.9）
- 割线（暖橙色，`always_redraw`）
- 隐形的切线（黄色虚线，opacity 0.0，稍后激活）
- 右上角实时差商数值（白色 `DecimalNumber`，2位小数）

**动画序列：**

**第一阶段：从右侧逼近**

1. Q 在 h=2.0 处出现（暖橙色）— 0.5s
2. 右上角出现差商数值框（白色小字 `当前斜率≈` + 数值）— 0.5s

3. Q 缓慢滑向 P（`ValueTracker`：h 从 2.0 → 0.5，用时 3.0s）
   - 割线随之旋转
   - 差商数值实时更新
   - 节奏：**从容，不急**

4. **`Wait(2.0)`** — 在 h=0.5 处暂停，割线仍与切线有明显夹角

5. Q 继续趋近（h 从 0.5 → 0.05，用时 2.0s）
   - 割线与切线差距越来越小
   - Q 点越来越小（暗示"快和P重合了"）

6. **`Wait(2.5)`** — **第一个核心停顿**，割线已经几乎是切线了

**第二阶段：从左侧逼近（证明两侧一致）**

7. Q 重置到 h=-2.0（左侧）— 0.5s
8. Q 从左侧缓慢趋近 P（h 从 -2.0 → -0.05，用时 3.5s）
   - 割线从另一个方向旋转过来
   - 差商数值从不同起点逐渐趋近同一个值

9. **`Wait(2.0)`** — 让观众确认"两侧趋向同一条线"

**第三阶段：切线登场**

10. Q 点和割线同时淡出（`FadeOut`，1.0s）
11. 黄色虚线（隐形的切线）从 opacity 0.0 → 1.0，同时变为实线（`animate`，1.5s）
    - 此刻加一个轻微的 `Flash`（黄色光晕，0.3s）——就像"咔嗒"一声锁定

12. **`Wait(3.0)`** — **全片最长停顿**，切线独自在屏幕上，什么都不说

**第四阶段：结论浮现**

13. 右上角的差商数值框淡出（1.0s）
14. 屏幕下方分两行出现（间隔1.5s，黄色，大字）：

    ```
    第一行（先出现）：  割线斜率  →  切线斜率
    第二行（1.5s后）：  平均变化率  →  瞬时变化率
    ```

15. `Wait(2.5)` — 让观众把两行建立对应关系

16. 两行文字用黄色边框框住，成为"结论框" — 0.8s
17. **`Wait(2.0)`**

**设计意图：**
全片的情绪高潮。这个场景有几个关键决策：
- **去掉一切不必要的标注**：只保留曲线、P、Q、割线。越简单，视觉事件越震撼。
- **从两侧逼近**：让观众自己发现"不管从哪边来，都趋向同一条线"，这是双侧极限的几何直觉。
- **切线的"咔嗒"锁定**：`Flash` 效果是情绪的释放点——终于，那条线"固定"了。
- **结论分两行、有间隔**：让观众有时间把"割线→切线"和"平均→瞬时"这两对对应起来。

### 场景 9：代数确认——把看到的写下来（55 秒）

**情绪目标：** 代数不是新任务，是对场景8的**翻译**

**布局：** 左三分之一保留切线图，右三分之二为推导区域

**视觉元素：**
- 左侧：场景8结尾的切线图（黄色切线 + 点P），降至 60% 透明度
- 右侧：逐步展开的代数推导

**动画序列：**

**锚定阶段（15秒）：**

1. 场景8结尾的画面保留，左侧切线图降至 60% — 0.8s

2. 右侧出现"翻译"引导（白色，`Write`，1.5s）：

   > **"刚才看到的，用代数写出来："**

3. 在右侧出现差商公式（`Write`，2.0s）：

   ```
   平均变化率 = [16(2+h)² - 16·4] / h
   ```

   - 写出时，左侧切线图中的P和Q（橙色，半透明）短暂出现并高亮 — 用颜色对应：分子的"16(2+h)²"对应Q点，"16·4"对应P点

4. `Wait(2.0)` — 确保观众理解公式从哪里来

**推导阶段（30秒，逐行展开）：**

5. 展开 `(2+h)²`（`TransformMatchingShapes`，1.5s）：

   ```
   = [64 + 64h + 16h² - 64] / h
   ```

6. 消去两个64（红色删除线划掉，1.0s）：

   ```
   = [64h + 16h²] / h
   ```

7. 除以 h（高亮 h，然后约分，1.5s）：

   ```
   = 64 + 16h       （当 h ≠ 0）
   ```

8. 左侧切线图中，Q点再次出现并开始趋近P（快速重播场景8的核心动画，1.5s）

9. 右侧出现 `h → 0` 的箭头，指向结果（`Write`，1.0s）：

   ```
   h → 0  →  64 + 16·0 = 64
   ```

   - 数字 `64` 放大，变黄色

10. `Wait(2.0)`

**确认阶段（10秒）：**

11. 左侧切线图完全亮起（回到 100% 透明度），切线高亮 `Flash` — 0.8s

12. 右侧出现结论（黄色框，`Write`，1.0s）：

    ```
    切线斜率 = 64
    ```

13. **`Wait(2.5)`**

**设计意图：**
"锚定阶段"是原分镜缺失的关键步骤。观众从场景8的视觉高潮进入代数推导时，如果没有引导，会感到"我们为什么突然开始算这个"。这里的"刚才看到的，用代数写出来"就是那座桥。

代数推导中，每一步都用**颜色**和**左侧图像**保持与几何直觉的连接。

### 场景 10：收束——命名与展望（25 秒）

**情绪目标：** 满足感 + 好奇心，带着期待离开

**布局：** 全屏，居中，简洁

**视觉元素：**
- 场景8的切线动画缩小后出现在左上角（视觉回忆）
- 中央：递进出现的三行文字

**动画序列：**

1. 场景8的切线动画（缩小至约1/4屏幕大小）出现在左上角（`FadeIn`，1.0s）
   - 这是"视觉回忆"，让观众和刚才的啊哈时刻建立联系

2. 中央出现第一行（白色，`Write`，1.0s）：

   ```
   我们一直在做的事：
   让 h → 0，观察结果趋向哪里
   ```

3. `Wait(1.5)`

4. 第二行出现，用黄色大字（`Write`，1.2s）：

   ```
   这件事有一个名字：极限
   ```

   - "极限"二字用 `FocusOn` 效果：短暂放大后回到正常大小，黄色

5. `Wait(1.5)`

6. 第三行出现（白色，小字，`FadeIn`，1.0s）：

   > **"下一节，我们来把这件事说得准确。"**

7. 左上角的缩小动画也在此时定格——切线静止，黄色，清晰

8. **`Wait(3.0)`** — 留白结束，让最后的画面自己说话

9. 全屏缓慢 `FadeOut` — 1.5s

**设计意图：**
结尾的设计有三个考量：
1. **视觉回忆**：把场景8缩小放在角落，让结尾和高潮产生情感联系
2. **命名**：不是"我们需要极限"（被动），而是"这件事有一个名字：极限"（发现感）
3. **"说得准确"**：比"学习极限"更吸引人——暗示这是一种精确化，而不是全新的内容

## 六、节奏检查清单

渲染完成后必须逐项确认：

**停顿时间检查**
- [ ] 场景1（除以零）：死局停顿 ≥ 3.0 秒
- [ ] 场景4（矛盾爆发）：结论停顿 ≥ 3.0 秒
- [ ] 场景5（逼近数列）：`→64` 停顿 ≥ 2.5 秒
- [ ] 场景8（割线→切线）：切线独自停顿 ≥ 3.0 秒，总停顿 ≥ 7.0 秒
- [ ] 场景9（代数确认）：差商公式出现后停顿 ≥ 2.0 秒

**信息密度检查**
- [ ] 任意一场暂停3秒后，观众能看懂当前画面
- [ ] 没有连续两个场景用相同类型的动画（如两个变形、两个表格）
- [ ] 公式出现之前，已有对应的几何/物理解释

**颜色语义检查**
- [ ] 没有元素用了红色但不表示"矛盾/问题"
- [ ] 割线全程为暖橙色，切线全程为黄色，两者从不混用

**叙事逻辑检查**
- [ ] 公式是**最后**出现的，它是对已经看到的东西的描述
- [ ] 场景3的伏笔（"结果依赖区间"）在场景4中被明确收回

## 八、设计哲学备忘

> **好的数学动画，不是让公式运动起来，**
> **而是让观众先看到几何事实，**
> **再感觉代数是必然的描述。**

三个核心判断标准：
1. 如果**静音**观看，观众能理解场景8的核心内容吗？
2. 公式出现的那一刻，是"终于明白它在说什么了"，还是"又来了一个需要记忆的公式"？
3. 场景8结束后，观众是感到"解决了一个问题"，还是"看到了更大的问题的开始"？

**答案都应该是前者，但第3题的正确答案是：两者都有。**
