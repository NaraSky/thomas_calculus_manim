# Thomas Calculus 2.2 — 函数的极限与极限定律
## Manim 动画分镜设计文档

> **核心问题（全片灵魂）**
> 我们在2.1节"看到"了极限——割线趋近切线。
> 但"趋近"到底是什么意思？能计算吗？能证明吗？

## 三、场景详细设计

### 场景 0：承接——我们已经"看到"了什么（15 秒）

**情绪目标：** 唤起上节的记忆，建立"接着讲"的节奏感，而不是从零开始

**布局：** 左侧：场景缩略图回忆；右侧：今日问题

**动画序列：**

1. 黑屏，从左侧滑入上一节结尾的画面（**切线图**，缩小至屏幕左三分之一）— 1.0s
   - 画面略带动感（轻微缓慢缩放），暗示"这是过去的"

2. 右侧出现两行文字（`Write`，间隔 0.8s）：

   ```
   上节：我们看到了"趋近"这件事。
   本节：我们来把它说清楚。
   ```

3. 底部出现本节标题（黄色，`FadeIn`，0.8s）：

   > **函数的极限与极限定律**

4. `Wait(1.5)` — 进入场景1

**设计意图：**
不从头解释，直接承接。观众看到上节的画面会有"熟悉感"，立刻进入状态。"我们来把它说清楚"比"今天学习极限的定义"更有驱动力——暗示这不是全新的东西，而是把已经直觉到的东西精确化。

### 场景 1：核心例子——带空洞的直线（50 秒）

**情绪目标：** 用一个极度简单的例子，让"极限不关心函数在该点的值"这件事变得显而易见

**布局：** 左侧：函数图像；右侧：数值逼近表格

**视觉元素：**
- 坐标系（标准，轴线白色）
- 直线 `y = x + 1`，但在 `x=1` 处有一个**红色空心圆**（hole）
- 右侧：两列数值表（`x` | `f(x)`）

**动画序列：**

**第一阶段：引入函数（20秒）**

1. 坐标系出现 — 0.8s

2. 右侧出现函数定义（`Write`，1.5s）：

   $$f(x) = \frac{x^2 - 1}{x - 1}$$

3. 屏幕下方出现一行注释（白色，小字）：

   > *"在 x=1 处，分母为零，f(1) 无定义。"*

4. `Wait(1.0)`

5. 左侧画出直线 `y = x + 1`（蓝白色，`Create`，1.5s）
   - 但在 `x=1` 处停下，留出空位

6. 在 `x=1` 处画出**红色空心圆**（`GrowFromCenter`，0.5s）
   - 这个空洞要足够大，清晰可见

7. `Wait(1.5)` — 让观众看清"直线上有个洞"

**第二阶段：从两侧逼近（30秒）**

8. 右侧出现数值表框架（两列标题：`x` | `f(x)`）— 0.5s

9. **从右侧（x > 1）逼近**，逐行出现（`LaggedStart`，间隔 0.6s）：

   | x | f(x) |
   |:--|:--|
   | 1.1 | 2.1 |
   | 1.01 | 2.01 |
   | 1.001 | 2.001 |

   - 每行出现时，左侧图中对应 x 位置画一个**暖橙色小点**，从右侧滑向空洞
   - 点越来越接近空洞，但永远不填入

10. **从左侧（x < 1）逼近**，继续追加（`LaggedStart`）：

    | x | f(x) |
    |:--|:--|
    | 0.9 | 1.9 |
    | 0.99 | 1.99 |
    | 0.999 | 1.999 |

    - 左侧图中，点从左侧滑向空洞

11. 所有趋近点停止后，`Wait(1.5)`

12. 两侧的暖橙色点同时闪烁（`Indicate`），然后在空洞正上方出现一个**黄色实心圆**（`GrowFromCenter`，0.8s），悬浮于空洞上方（稍微偏上一点点，视觉上"悬停"而非"填入"）

13. 右侧表格右侧出现箭头 `→ 2`（黄色）— 0.8s

14. `Wait(2.0)`

15. 屏幕下方出现极限记号（黄色，`Write`，1.5s）：

    $$\lim_{x \to 1} \frac{x^2-1}{x-1} = 2$$

16. **`Wait(2.5)`** — 关键停顿

**设计意图：**
这个例子之所以完美，是因为它极其简单（就是一条直线加一个洞），但它完整地展示了极限的本质：**我们关心的是"趋近时发生什么"，而不是"那个点上是什么"。**

黄色悬浮点刻意不填入空洞，就是在说：极限值 `2` 是函数"想去的地方"，但函数本身没有在那里定义。

### 场景 2：三张面孔——极限与函数值的三种关系（40 秒）

**情绪目标：** 建立准确的认知：极限 ≠ 函数值，但也可以等于；函数值甚至可以不存在

**布局：** 三个并排的小坐标系（各占屏幕三分之一）

**视觉元素：**
三个函数，都在 x→1 时极限为2：
- 图(a)：`f(x) = (x²-1)/(x-1)`，x=1处**空心红圆**（无定义）
- 图(b)：`g(x)` 同上但 `g(1)=1`（**白色实心点在 y=1**，空心圆在 y=2）
- 图(c)：`h(x) = x+1`，x=1处**实心点在 y=2**（极限等于函数值）

**动画序列：**

1. 三个坐标系同时出现（`FadeIn`，1.0s）

2. 三个图像同时绘制（`Create`，2.0s）
   - 颜色统一为蓝白色
   - 关键点用颜色区分：空心红圆 / 白色实心点（偏离）/ 白色实心点（一致）

3. 三图下方分别出现标注（`Write`，各 0.8s，`LaggedStart`）：
   - 图(a)下：**`f(1)` 无定义**（红字）
   - 图(b)下：**`g(1) = 1 ≠ 2`**（橙字）
   - 图(c)下：**`h(1) = 2 ✓`**（绿字）

4. `Wait(1.5)`

5. 在三图**上方**同时出现同一个极限（黄色）：

   $$\lim_{x \to 1} = 2$$

   用三条黄色箭头分别指向三个图的极限位置（悬浮黄点）— 1.5s

6. **`Wait(2.5)`**

7. 用黄色边框框住三图，出现总结文字（`Write`，1.5s）：

   > **极限只关心"趋近"，不关心"到达"。**

8. **`Wait(2.5)`** — 关键停顿，这是本节的第一个核心认知

**设计意图：**
三张图并排是经典的教学设计，但关键是**上方的同一个极限记号 + 三条箭头**——在视觉上强调"这三个函数在极限这件事上是相同的"，而不是让观众自己去对比三张图的差异。差异是次要的，相同才是重点。

### 场景 3：极限不存在——三种死法（45 秒）

**情绪目标：** 了解边界，建立完整认知——并非所有函数在所有点都有极限

**布局：** 三个并排的小坐标系 + 下方文字说明

**视觉元素：**
三种典型的"极限不存在"：
- 图(a)：单位阶跃函数，`x→0` 时左右极限不等（跳跃间断）
- 图(b)：`g(x) = 1/x`，`x→0` 时趋向无穷（无界）
- 图(c)：`f(x) = sin(1/x)`，`x→0` 时无限振荡

**动画序列：**

**每张图用"问题动画"来展示，而不是直接静态呈现：**

1. 图(a)——跳跃（15秒）：
   - 先画左半边图像（`x<0` 时值为0）
   - 从左侧滑来一个蓝色滑块（"趋近 x=0 中的探针"）
   - 探针到达 x=0 时，函数值 = 0
   - 然后从右侧滑来另一个蓝色探针
   - 右侧探针到达 x=0 时，函数值 = 1
   - 两个探针同时显示，数值不同，用红色 `≠` 连接 — 0.8s
   - 下方标注：**"左右极限不相等"**（红字）— 0.5s

2. 图(b)——无界（15秒）：
   - 画出 `1/x` 的正半轴部分
   - 一个蓝色探针从右侧趋近 x=0
   - 函数值实时显示（`DecimalNumber`），从 `1` 飙升到 `100`，`1000`，`...`
   - 最终显示 `→ ∞`（红色）— 1.0s
   - 下方标注：**"函数值趋向无穷"**（红字）— 0.5s

3. 图(c)——振荡（15秒）：
   - 画出 `sin(1/x)` 在 x>0 时的图像（靠近原点处振荡密集）
   - 蓝色探针从右侧趋近，对应的函数值在 `+1` 和 `-1` 之间来回跳动
   - 用红色区间标注 `[-1, +1]`，表示函数值无法收敛到单一数值 — 1.0s
   - 下方标注：**"函数无限振荡"**（红字）— 0.5s

4. 三图都展示完后，`Wait(2.0)`

5. 三图上方出现统一的总结（红色，`Write`，1.5s）：

   > **极限不存在的三种原因：跳跃 · 无界 · 振荡**

6. **`Wait(2.5)`**

**设计意图：**
"探针"动画是本节的新视觉语言——用一个可以滑动的探针来模拟"x趋近c"这个过程，比静态图像更有说服力。观众会感受到"函数本身在阻止极限存在"，而不只是"结果不对"。这三种情形对应的是教材例4，但视觉化的方式完全不同。

### 场景 4：极限的正式定义——精确化（35 秒）

**情绪目标：** 定义感觉像是对已经看到的事情的"精确翻译"，而不是新的内容

**布局：** 上方：视觉直觉图；下方：正式定义框

**视觉元素：**
- 上方：带"空洞"的函数图，极限点 `L`，ε-带（黄色水平带），δ-带（蓝色竖直带）
- 下方：定义框

**动画序列：**

**第一阶段：从直觉到符号（25秒）**

1. 复用场景1的函数图（`f(x) = (x²-1)/(x-1)`）— 0.5s

2. 在 `y=2` 处画一条黄色水平线，然后在其上下画 ε 宽度的黄色阴影带（`FadeIn`，1.0s）
   - 标注 `L+ε`（上边界）和 `L-ε`（下边界）

3. 旁边出现提示（白色小字，`Write`，0.8s）：

   > *"ε 是你要求的精度：f(x) 必须落在这个带里。"*

4. `Wait(1.5)`

5. 在 `x=1` 处画蓝色竖直阴影带（`FadeIn`，1.0s）
   - 标注 `c-δ`（左边界）和 `c+δ`（右边界）

6. 旁边出现提示（白色小字）：

   > *"δ 是你控制的范围：只要 x 在这个带里……"*

7. 高亮演示：当 x 落在蓝色带内（蓝色），f(x) 落在黄色带内（黄色）— 1.0s

8. `Wait(1.5)`

**第二阶段：正式定义（10秒）**

9. 屏幕下方出现定义框（黄色边框，`Create`，0.5s）

10. 定义逐行出现（`Write`，每行 1.0s）：

    ```
    lim f(x) = L
    x→c

    对每个 ε > 0，存在 δ > 0，使得：
    0 < |x - c| < δ  时，有  |f(x) - L| < ε
    ```

    - `ε` 用黄色，`δ` 用蓝色，与图中颜色对应
    - `0 < |x - c|` 用橙色标注：这是"x不等于c"的关键条件

11. `Wait(2.5)`

12. 圈出 `0 < |x - c|` 部分（橙色 `Circumscribe`），出现注释（白色小字）：

    > *"0 < 是关键：我们不关心 f(c) 本身是什么。"*

13. **`Wait(2.5)`**

**设计意图：**
ε-δ 定义是本节最抽象的内容，但如果在视觉上先建立"ε 带"和"δ 带"的直觉，定义里的每个符号都会感觉像是"对图中那个阴影带的命名"。`0 < |x-c|` 的橙色高亮是关键——它对应场景1中黄色悬浮点"不填入空洞"的那个视觉决策。

### 场景 5：两个基础极限——工具箱的地基（25 秒）

**情绪目标：** 快速建立最简单的两块积木，为极限定律做铺垫

**布局：** 左右两个图，各占半屏

**视觉元素：**
- 左图：恒等函数 `f(x) = x`（一条过原点的斜线），在 x=c 处趋近
- 右图：常数函数 `f(x) = k`（一条水平线），在 x=c 处趋近

**动画序列：**

1. 两图同时出现（`FadeIn`，0.8s）

2. **左图动画**（恒等函数）：
   - 蓝色探针从右侧趋近 x=c
   - 函数值同步显示，最终等于 c
   - 出现极限记号（黄色）：$\lim_{x \to c} x = c$（1.0s）

3. **右图动画**（常数函数）：
   - 蓝色探针趋近 x=c（任意 c）
   - 函数值始终为 k，完全不变
   - 出现极限记号（黄色）：$\lim_{x \to c} k = k$（1.0s）

4. `Wait(1.5)`

5. 两图下方各出现一行直觉解释（白色小字，`FadeIn`）：
   - 左图下："趋近谁，就等于谁。"
   - 右图下："常数不会变，无论如何趋近。"

6. **`Wait(2.0)`**

7. 屏幕上方出现提示（浅紫色，`Write`，1.0s）：

   > **这是极限运算的两块砖。接下来，我们用它们搭房子。**

8. `Wait(1.0)`

**设计意图：**
这两个极限本身没有悬念，重要的是**建立"工具箱"的感受**。结尾的"两块砖"比喻预告了接下来的极限定律——观众会带着期待进入场景6。

### 场景 6：极限定律——工具箱扩充（60 秒）

**情绪目标：** 定律不是需要记忆的规则，而是"显然成立"的事实；工具箱的建立应该有节奏感

**布局：** 中央：主演示区；右侧：逐渐建立的"工具箱"（浅紫色定律卡片列表）

**视觉元素：**
- 中央：直观演示每条定律成立的几何原因
- 右侧：浅紫色圆角矩形卡片，逐条出现，最终形成完整工具箱
- 右侧工具箱标题：**"极限定律"**（浅紫色，带边框）

**动画序列：**

**引入（5秒）：**

1. 屏幕中央出现两个已知条件（白色，`Write`，1.0s）：

   $$\lim_{x \to c} f(x) = L \quad \lim_{x \to c} g(x) = M$$

2. 右侧出现空白的"工具箱"框架（浅紫色边框，标题"极限定律"）— 0.8s

**逐条定律演示（每条约7秒，共5条主要定律）：**

**定律1：和法则（8秒）**

3. 中央出现两条曲线 `f(x)` 和 `g(x)`（蓝绿色），各自趋近 `L` 和 `M` — 0.8s

4. 两条曲线"叠加"，变为 `f(x)+g(x)` 趋近 `L+M`（动画：两条曲线在纵轴方向合并）— 1.5s

5. 直觉注释（白色小字）：

   > *"如果两个量各自在趋近，它们的和自然也在趋近。"*

6. 右侧工具箱新增第一张卡片（`GrowFromEdge`，浅紫色）：

   $$\lim(f + g) = L + M$$

7. `Wait(1.0)`

**定律2：积法则（8秒）**

8. 演示：两个趋近量相乘（用面积类比——两条长度趋近L和M的线段，围成的矩形面积趋近LM）— 2.0s

9. 直觉注释：

   > *"两个'几乎是L'和'几乎是M'的量，相乘后几乎是LM。"*

10. 右侧工具箱新增（`GrowFromEdge`，浅紫色）：

    $$\lim(f \cdot g) = L \cdot M$$

**定律3：常数倍法则（5秒）**

11. 简短演示：曲线垂直拉伸 k 倍 — 1.0s

12. 工具箱新增：$\lim(k \cdot f) = k \cdot L$（`GrowFromEdge`）

**定律4：商法则（8秒）**

13. 演示：关键条件 `M ≠ 0`（用红色标注分母不能趋近零）— 1.0s

14. 演示：分子趋近 L，分母趋近 M（非零），商趋近 L/M — 1.0s

15. 直觉注释：

    > *"唯一的危险：分母趋近零。那是下一场景要处理的。"*

16. 工具箱新增：$\lim(f/g) = L/M \quad (M \neq 0)$

**定律5：幂法则 + 根式法则（8秒）**

17. 快速演示：积法则反复应用 → 幂法则；逆运算 → 根式法则 — 1.5s

18. 工具箱依次新增两张卡片（`LaggedStart`）：

    $$\lim[f(x)]^n = L^n$$
    $$\lim \sqrt[n]{f(x)} = \sqrt[n]{L}$$

**完整工具箱亮相（8秒）：**

19. 所有定律卡片同时亮起（`Indicate`，浅紫色脉冲）— 1.0s

20. 中央出现一道"综合题"（`Write`，1.5s）：

    $$\lim_{x \to c} (x^3 + 4x^2 - 3)$$

21. 拆解动画：把这个极限一步步分解成刚才的工具（逐步用彩色框框出每个子限）— 2.5s

22. 最终结果：直接代入 c，得到 `c³ + 4c² - 3`（`Write`，1.0s）

23. **`Wait(2.5)`**

**设计意图：**
极限定律的关键不是内容（它们都显然成立），而是**节奏**——每条定律配一个简短的几何直觉，然后立刻加入右侧工具箱。观众在场景结束时应该感到"我有了一套工具"，而不是"我背了六条规则"。

商法则的"M≠0危险信号"为下一个场景做铺垫。

### 场景 7：多项式与有理函数——直接代入（30 秒）

**情绪目标：** "哦，大多数情况下其实很简单"——在复杂之前给一个容易的喘息

**布局：** 居中，公式推导为主

**视觉元素：**
- 多项式极限定理（定理框）
- 有理函数极限定理（定理框）
- 一个具体例子的计算过程

**动画序列：**

1. 屏幕上方出现标题（白色，`Write`，0.8s）：

   > **对多项式和有理函数，极限可以直接代入。**

2. 出现多项式定理框（黄色边框，`Create`，0.8s）：

   $$\lim_{x \to c} P(x) = P(c)$$

   - 旁边注释（白色小字）：*"多项式在任意点都可以直接代入。"*

3. 出现有理函数定理框（黄色边框，`Create`，0.8s）：

   $$\lim_{x \to c} \frac{P(x)}{Q(x)} = \frac{P(c)}{Q(c)} \quad (Q(c) \neq 0)$$

   - 红色高亮 `Q(c) ≠ 0`

4. `Wait(1.5)`

5. 演示一个例子（`Write`，逐步展示，2.0s）：

   $$\lim_{x \to -1} \frac{x^3 + 4x^2 - 3}{x^2 + 5} = \frac{(-1)^3 + 4(-1)^2 - 3}{(-1)^2 + 5} = \frac{0}{6} = 0$$

   - 代入步骤用暖橙色高亮，结果用黄色

6. `Wait(1.5)`

7. 屏幕角落出现小字（白色，30%透明度，预告）：

   > *"但如果 Q(c) = 0 呢？"*

8. **`Wait(2.0)`**

**设计意图：**
这个场景故意要快，因为内容本身没有悬念。重要的是结尾的那行小字——它再次制造了"还没结束"的悬念，把观众推向下一个场景。

### 场景 8：零分母——约分救场（55 秒）

**情绪目标：** "堵死的路，其实有一个出口"——代数操作在关键时刻的力量

**布局：** 左侧：函数图（含空洞）；右侧：代数推导

**视觉元素：**
- 函数 `f(x) = (x²+x-2)/(x²-x)`
- 左侧：函数图（在 x=1 处有空洞）
- 右侧：约分过程

**动画序列：**

**第一阶段：遭遇障碍（15秒）**

1. 右侧出现极限问题（`Write`，1.0s）：

   $$\lim_{x \to 1} \frac{x^2 + x - 2}{x^2 - x}$$

2. 尝试直接代入（暖橙色，`Write`，1.0s）：

   $$= \frac{1 + 1 - 2}{1 - 1} = \frac{0}{0}$$

3. 用红色 `Circumscribe` 圈住 `0/0`（1.0s）

4. 屏幕旁边出现红色文字（`Write`，0.8s）：

   > **0/0 = 无意义！**

5. `Wait(1.5)` — 让观众感到"堵死了"

**第二阶段：寻找出路（30秒）**

6. 右侧出现分析提示（白色，`Write`，1.0s）：

   > *"分子和分母在 x=1 处都等于零，说明它们有公因子 (x-1)。"*

7. 分子分解（逐步展开，每步 1.0s，颜色标注公因子）：

   $$x^2 + x - 2 = (x-1)(x+2)$$

8. 分母分解：

   $$x^2 - x = x(x-1)$$

9. 原式改写，公因子高亮（暖橙色框住两个 `(x-1)`）：

   $$\frac{(x-1)(x+2)}{x(x-1)}$$

10. **约分动画**：两个 `(x-1)` 同时画上删除线，消失 — 1.0s

11. 结果出现（`Write`，1.0s）：

    $$= \frac{x+2}{x} \quad (x \neq 1)$$

12. 关键注释（白色，`Write`，0.8s）：

    > *"`x ≠ 1` 时，两个表达式完全相同。*
    > *极限只关心趋近，不关心 x=1 时的值。"*

13. `Wait(1.5)`

**第三阶段：代入求值（10秒）**

14. 继续计算（`Write`，1.0s）：

    $$\lim_{x \to 1} \frac{x+2}{x} = \frac{1+2}{1} = 3$$

15. 左侧图中，原函数的图像出现（含空洞），同时在 x=1 的正上方标出黄色悬浮点（y=3）— 1.0s

16. 右侧结论框（黄色，`Create`，0.8s）：

    $$\lim_{x \to 1} \frac{x^2 + x - 2}{x^2 - x} = 3$$

17. **`Wait(2.5)`**

**设计意图：**
这个场景的戏剧结构是：**堵死 → 侦察 → 发现出口 → 突破**。"约分动画"（两个因子同时被划掉）是本场景的视觉高潮，应该干脆利落，像是"打开了一扇门"。

左侧图像在最后才出现并显示黄色悬浮点，是为了让"几何解释"成为代数推导的验证，而不是前提。

### 场景 9：夹逼定理——优雅的间接方法（55 秒）

**情绪目标：** 数学中最优雅的策略之一——你不需要直接计算，只需要把目标"夹住"

**布局：** 全屏，以视觉为主

**视觉元素：**
- 三条曲线：`g(x)`（蓝色，下界）、`f(x)`（暖橙色，目标）、`h(x)`（绿色，上界）
- 两条界曲线之间的阴影区域（表示"夹逼带"）
- 定理框

**动画序列：**

**第一阶段：问题设置（15秒）**

1. 屏幕上方出现问题（`Write`，1.0s）：

   > **有时候，f(x) 的极限很难直接算。**
   > **但如果我们能"夹住"它呢？**

2. 先画中间的目标曲线 `f(x)`（暖橙色，`Create`，1.5s）
   - 这条曲线在 x=c 附近很复杂（例如 `sin(1/x)` 乘以某个因子），难以直接计算极限

3. `Wait(1.0)`

**第二阶段：建立夹逼（20秒）**

4. 从下方"升起"曲线 `g(x)`（蓝色，`Create`，1.0s），始终低于 `f(x)`

5. 从上方"降落"曲线 `h(x)`（绿色，`Create`，1.0s），始终高于 `f(x)`

6. 在 `g`、`h` 之间填充轻微阴影（`FadeIn`，0.8s）
   - 这个阴影带就是"夹逼区域"，`f(x)` 被困在里面

7. `Wait(1.0)`

8. **关键动画**：`g` 和 `h` 的曲线同时向 x=c 处的同一个 y 值 `L` 收拢 — 2.5s
   - 阴影带越来越窄
   - `f(x)` 也被迫收拢到同一点

9. 在收拢的最后，三条曲线同时到达 `(c, L)`（黄色大点，`Flash`）— 0.5s

10. **`Wait(2.0)`** — 让观众感受到"f(x) 被逼到那里了"

**第三阶段：定理正式表述（10秒）**

11. 屏幕下方出现定理框（黄色边框，`Create`，0.5s），逐行填入（`Write`，每行 0.8s）：

    ```
    定理（夹逼定理）：

    若 g(x) ≤ f(x) ≤ h(x)
    且 lim g(x) = lim h(x) = L
    则 lim f(x) = L
    ```

    - `g(x)` 用蓝色，`h(x)` 用绿色，`f(x)` 用暖橙色，`L` 用黄色

12. `Wait(2.0)`

**第四阶段：经典应用（10秒）**

13. 屏幕右侧出现一个应用例子（`Write`，1.5s）：

    $$\lim_{\theta \to 0} \sin\theta = 0$$

    - 用夹逼：$-|\theta| \leq \sin\theta \leq |\theta|$
    - 三条线在 θ=0 处交于 0

14. 出现结论（黄色，`Write`，1.0s）：

    > **不需要直接计算——夹住了，就到了。**

15. **`Wait(2.5)`**

**设计意图：**
夹逼定理的核心视觉是"阴影带收窄"——这是最直观的展示方式。特别是"g 和 h 从两侧收拢，f 被迫到达同一点"这个动画，应该配合`Flash`效果，让那个汇聚的瞬间有情绪释放感。

这个场景在节奏上要比场景6-8慢一点，因为夹逼定理本身有一种"优雅的停顿感"。

### 场景 10：收束与展望（25 秒）

**情绪目标：** 总结"工具箱"，同时激发对2.3节的期待

**布局：** 居中，全屏

**动画序列：**

1. 屏幕上方出现本节的"收获清单"（`LaggedStart`，每行 0.8s）：

   ```
   本节我们有了：

   ✓ 极限的定义（说清楚了"趋近"）
   ✓ 三种极限不存在的情形
   ✓ 极限定律（七条运算规则）
   ✓ 约分处理零分母
   ✓ 夹逼定理（间接计算的策略）
   ```

   - 每行的 `✓` 为黄色，内容为白色
   - 极限定律那行的文字为浅紫色

2. `Wait(2.0)`

3. 屏幕下方出现铺垫（白色，`Write`，1.5s）：

   > **但"定义"还是不够精确。**
   > **"对每个 ε，存在 δ"——这到底怎么用？**

4. 最后一行出现（黄色，大字，`FadeIn`，1.0s）：

   > **下一节：极限的精确定义。**

5. **`Wait(2.0)`**

6. 全屏缓慢 `FadeOut` — 1.5s

**设计意图：**
收获清单不是为了"总结"，而是为了让观众感受到"我真的拿到了什么"。结尾的"但还不够精确"是诚实的——ε-δ 在场景4只是直觉介绍，真正的形式化在2.3节。这个"未完成感"是好事，让观众带着问题进入下一节。

## 九、设计哲学备忘

> **2.2节的挑战不同于2.1节。**
> 2.1节是"从零建立直觉"；
> 2.2节是"在直觉上搭建工具"。
>
> 工具必须感觉像是**你在掌控它**，
> 而不是**它在限制你**。

三个核心判断标准：
1. 看完场景6（极限定律），观众是感到"有了一套工具"，还是"记了一堆规则"？
2. 场景8（约分）的"约分动画"是否足够干脆——就像"打开了一扇门"？
3. 场景9（夹逼定理）是否让观众感受到"数学也可以是优雅的间接"？
