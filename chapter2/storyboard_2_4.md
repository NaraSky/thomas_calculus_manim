# Thomas Calculus 2.4 — 单侧极限
## Manim 动画分镜设计文档

> **核心问题（全片灵魂）**
> 有些函数在某一点的左右两侧行为截然不同。
> 当我们说"趋近"，我们是从哪个方向来的？

## 二、视觉设计规范

### 颜色语义（继承前三节，新增两色）

| 颜色 | 色值 | 语义 |
|:--|:--|:--|
| 冰蓝 | `#58C4DD` | x轴 / 横向趋近 / δ |
| 草绿 | `#83C167` | y轴 / 函数值 / 纵向 |
| 亮黄 | `#FFFF00` | 极限值 / 结论 / 切线 |
| 暖橙 | `#FF9D00` | 趋近过程 / 探针 |
| 亮红 | `#FF4455` | 矛盾 / 不存在 / 跳跃 |
| 浅紫 | `#C084FC` | 极限定律（继承2.2）|
| 天青 | `#5EEAD4` | δ的响应 / 应答方（继承2.3）|
| 珊瑚 | `#FB923C` | ε的挑战 / 挑战方（继承2.3）|
| 玫红 | `#F472B6` | 左极限 / 从左趋近（**新增**）|
| 薄荷 | `#34D399` | 右极限 / 从右趋近（**新增**）|
| 浅白 | `#EEEEEE` | 普通文字 / 坐标轴 |
| 深灰 | `#1C1C2E` | 背景 |

> **新增颜色说明：**
> 本节引入方向性：左极限用**玫红色**，右极限用**薄荷色**。
> 两色在视觉上形成"左边/右边"的直觉对应——玫红偏暖，薄荷偏冷，
> 天然契合"从不同温度的方向趋近"的感受。
> 当左右极限相等时，两色**融合为黄色**（极限存在的视觉语言）。

### 本节核心视觉符号

| 符号 | 视觉表示 | 含义 |
|:--|:--|:--|
| 左向探针 | 玫红色箭头，向右滑动 | 从左侧（x < c）趋近 |
| 右向探针 | 薄荷色箭头，向左滑动 | 从右侧（x > c）趋近 |
| 双色汇聚点 | 玫红+薄荷 → 黄色 | 左右极限相等，双侧极限存在 |
| 双色分叉点 | 玫红、薄荷各停一处 | 左右极限不同，双侧极限不存在 |
| 单位圆扇形 | 白色单位圆 + 三色面积 | sin θ/θ 几何证明的核心图示 |

### 场景 0：承接——精确了，但还不够用（15 秒）

**情绪目标：** 建立"工具还不够用"的不安，但不是威胁感，而是好奇感

**布局：** 左侧：2.3节 ε-δ 带缩略图；右侧：今日引入的现象

**动画序列：**

1. 黑屏，左侧滑入2.3节的 ε-δ 带图（半透明，缩小版）— 1.0s

2. 右侧逐行出现（`Write`，间隔 0.8s）：

   ```
   上节：我们精确定义了"趋近"。
   但"趋近"还有一个问题没问：

   从哪个方向来？
   ```

3. 最后一行用黄色高亮（`Write`，0.8s）：

   > **从左，还是从右？**

4. 底部出现标题（黄色，`FadeIn`，0.8s）：

   > **单侧极限**

5. `Wait(1.5)`

**设计意图：**
开场不重复ε-δ内容，直接点出新维度。"从左还是从右"这个问题非常直觉化——观众会立刻想到"这有区别吗"，带着问题进入场景1。

### 场景 1：阶跃函数——左右真的不同（40 秒）

**情绪目标：** 用一个无可辩驳的具体例子，让观众亲眼看到"方向决定一切"

**布局：** 全屏坐标系，单焦点

**视觉元素：**
- 坐标系，函数 $f(x) = x/|x|$（$x<0$ 时为 $-1$，$x>0$ 时为 $1$）
- 玫红色左向探针（从左侧趋近 x=0）
- 薄荷色右向探针（从右侧趋近 x=0）
- x=0 处：两个颜色不同的悬浮点（分别在 y=-1 和 y=1）

**动画序列：**

**建立函数（10秒）：**

1. 坐标系出现 — 0.8s

2. 分两段绘制函数（`Create`，各 0.8s）：
   - x<0 的部分（y=-1，玫红色实线）从左向右绘制
   - x>0 的部分（y=1，薄荷色实线）从右向左绘制
   - 两段在 x=0 处均为**空心点**（红色）

3. `Wait(1.0)` — 让观众感受到"中间有个跳跃"

**左侧趋近（10秒）：**

4. 玫红色左向探针出现在 x=-2 处（箭头指向右，即趋近x=0的方向）— 0.5s

5. 探针缓慢向右滑动（x 从 -2 → -0.1，2.0s）
   - 同时：左侧下方显示 **"f(x) → -1"**（玫红色数字实时跟踪）

6. 探针在 x=-0.1 处停止，x=0 下方玫红色空心圆出现（y=-1）— 0.5s

7. 屏幕左侧弹出标注（玫红色，`Write`，0.8s）：

   $$\lim_{x \to 0^-} f(x) = -1$$

8. `Wait(1.0)`

**右侧趋近（10秒）：**

9. 薄荷色右向探针出现在 x=2 处（箭头指向左）— 0.5s

10. 探针向左滑动（x 从 2 → 0.1，2.0s）
    - 同时：右侧下方显示 **"f(x) → 1"**（薄荷色数字实时跟踪）

11. 探针在 x=0.1 处停止，x=0 上方薄荷色空心圆出现（y=1）— 0.5s

12. 屏幕右侧弹出标注（薄荷色，`Write`，0.8s）：

    $$\lim_{x \to 0^+} f(x) = 1$$

13. `Wait(1.0)`

**对比停顿（10秒）：**

14. 两个探针同时在屏幕上，玫红色停在下方，薄荷色停在上方 — `Wait(1.0)`

15. 屏幕中央出现对比线（红色水平虚线，在 y=-1 和 y=1 之间）— 0.5s

16. 红色大字出现（`Write`，1.0s）：

    > **-1 ≠ 1**

17. 下方出现结论（白色，`Write`，1.0s）：

    > **左右极限不同，因此双侧极限不存在。**

18. **`Wait(3.0)`** — 关键停顿，让这个不对等的视觉悬在那里

**设计意图：**
两个探针从不同方向同时存在于屏幕上，是本节最重要的视觉符号。玫红色停在下方、薄荷色停在上方，形成的色彩分叉感是"极限不存在"的直觉形象。`-1 ≠ 1` 出现的那一刻应该有一种"当然不对等"的不舒适感。

### 场景 2：单侧极限的正式定义（35 秒）

**情绪目标：** 定义是对场景1直觉的精确翻译，观众应该感到"这正是我刚才看到的"

**布局：** 上方：场景1的双探针图（保留，降至50%）；下方：双定义框

**动画序列：**

**左极限定义（15秒）：**

1. 场景1画面降至 50% 透明度 — 0.8s

2. 屏幕下方左侧出现定义框（玫红色边框，`Create`，0.5s），逐行填入（`Write`，间隔 0.8s）：

   ```
   左极限（从左趋近）：

   lim f(x) = L
   x→c⁻

   若对每个 ε > 0，存在 δ > 0，使得：
   c - δ < x < c  时，有  |f(x) - L| < ε
   ```

   - 写 `x→c⁻` 时，上方玫红色探针高亮并闪烁
   - 写 `c - δ < x < c` 时，上方左侧 δ 带（单侧）高亮（玫红色）
   - `c⁻` 用玫红色

3. 旁边出现注释（玫红色小字）：

   > *"`c⁻` 表示从左侧趋近——只考虑 x < c 的情形。"*

4. `Wait(1.0)`

**右极限定义（15秒）：**

5. 屏幕下方右侧出现定义框（薄荷色边框，`Create`，0.5s），填入（`Write`，间隔 0.8s）：

   ```
   右极限（从右趋近）：

   lim f(x) = L
   x→c⁺

   若对每个 ε > 0，存在 δ > 0，使得：
   c < x < c + δ  时，有  |f(x) - L| < ε
   ```

   - 写 `x→c⁺` 时，上方薄荷色探针高亮
   - 写 `c < x < c + δ` 时，上方右侧 δ 带（单侧）高亮（薄荷色）
   - `c⁺` 用薄荷色

6. **并排停顿**（两个定义框同时在屏幕上）— `Wait(1.5)`

7. 关键差异高亮：
   - 左定义中 `c - δ < x < c` 用黄色框出
   - 右定义中 `c < x < c + δ` 用黄色框出
   - 两者之间出现注释（白色）：**"区别只在 x 在 c 的哪一侧"**

8. **`Wait(2.5)`**

**设计意图：**
双定义框并排，差异只在x的范围（左侧还是右侧）。黄色框出关键差异后，观众应该立刻感受到"这两个定义几乎一样，只是方向不同"——这种"几乎相同"的感受让定义显得自然，而不是人为复杂化。

### 场景 3：连接定理——左右合体的条件（45 秒）

**情绪目标：** 建立"两侧合体"的视觉直觉，然后把它写成定理

**布局：** 全屏，以动画为主，定理框在最后出现

**视觉元素：**
- 坐标系，一条函数曲线
- 玫红色探针（左侧）、薄荷色探针（右侧）
- 汇聚点（融合动画）

**动画序列：**

**第一幕：两侧相等，汇聚成功（20秒）**

1. 函数曲线出现（连续，无跳跃，`Create`，1.5s）
   - 例：`f(x) = x + 1`，在 x=2 处有空心点（极限存在但函数值可能不同）

2. 玫红色探针从左侧出发 — 0.5s
3. 薄荷色探针从右侧出发 — 0.5s

4. 两个探针**同时**向 x=2 趋近（`animate`，2.5s）
   - 玫红色探针对应的函数值：→ 3
   - 薄荷色探针对应的函数值：→ 3
   - 两色值标注实时更新，均趋近 3

5. 两个探针几乎到达 x=2，停止 — `Wait(1.0)`

6. **汇聚动画**：玫红色点和薄荷色点在 y=3 处向中心靠拢，**融合为一个黄色大点**（`Transform`，1.0s）

7. 黄色大点出现时加 `Flash`（黄色光晕）— 0.3s

8. 屏幕上方出现（黄色，`Write`，1.0s）：

   $$\lim_{x\to 2} f(x) = 3$$

9. **`Wait(2.0)`**

**第二幕：两侧不等，无法汇聚（12秒）**

10. 画面切换：新的函数图（阶跃函数，跳跃发生在 x=1）— 1.0s

11. 两探针同时趋近 x=1：
    - 玫红色探针：函数值 → 0（停在下方）
    - 薄荷色探针：函数值 → 1（停在上方）

12. 两色点尝试汇聚，但方向相反，无法融合 — 用动画表现"两点各自停住，中间出现红色×号"（`FadeIn`，0.8s）

13. 屏幕上方出现（红色，`Write`，0.8s）：

    > **极限不存在（左右极限不相等）**

14. `Wait(1.0)`

**定理浮现（13秒）：**

15. 第一幕、第二幕画面淡化至 30% 透明度 — 0.8s

16. 屏幕中央出现定理框（黄色边框，`Create`，0.5s），填入（`Write`，间隔 0.8s）：

    ```
    定理：双侧极限的充要条件

    lim f(x) = L
    x→c

    当且仅当：

    lim f(x) = L   且   lim f(x) = L
    x→c⁻                x→c⁺
    ```

    - `lim f(x)` 用白色
    - 左极限记号用玫红色，右极限记号用薄荷色
    - 两个 `L` 用黄色

17. `Wait(1.5)`

18. 用箭头连接三个极限记号，形成"三角关系"（`GrowArrow`，0.8s）：

    - 左极限 → 双侧极限（薄箭头）
    - 右极限 → 双侧极限（薄箭头）
    - 注：**且两个单侧极限的值相同**（黄色强调）

19. **`Wait(2.5)`**

**设计意图：**
"汇聚动画"（玫红+薄荷→黄色）是本节最重要的视觉语言。两色融合为黄色，直觉上就是"左右相等 = 双侧极限存在"。这个动画应该有明显的"成功感"，对比第二幕的红色"×"失败符号，形成强烈的情感落差。

### 场景 4：综合例题——五个点的极限分析（50 秒）

**情绪目标：** "我现在能读懂一张复杂的函数图了"——建立分析图像的技能感

**布局：** 左侧：复杂函数图；右侧：逐点分析表

**视觉元素：**
- 左侧：一张包含多种特征的函数图（对应教材图2.27）
  - x=0 处：左侧无定义（左端点），右极限为1，函数值1
  - x=1 处：跳跃（左极限0，右极限1，函数值1）
  - x=2 处：空洞（极限1，函数值2）
  - x=3 处：连续（极限2，函数值2）
  - x=4 处：右侧无定义（右端点），左极限1，函数值不同
- 右侧：5行分析表（逐行出现）

**动画序列：**

**建立函数图（8秒）：**

1. 坐标系出现，函数图逐段绘制（`Create`，2.0s）
   - 颜色：主曲线白色，空心点红色边框，实心点白色填充
   - 关键点均标注坐标

2. `Wait(1.5)` — 让观众先整体看图

**逐点分析（每点约7秒，共5点，35秒）：**

**x=0 处（7秒）：**

3. 黄色圆圈在 x=0 处出现（`Circumscribe`，0.8s）

4. 右侧第1行出现（玫红色小字）：

   ```
   x=0（左端点）：
   左极限：不存在（函数在左侧无定义）
   右极限：lim⁺ f(x) = 1  ✓
   极限：lim f(x) = 1
   ```

   - 薄荷色探针在函数图上从右侧趋近 x=0，函数值 → 1（1.5s）

5. `Wait(0.8)`

**x=1 处（7秒）：**

6. 黄色圆圈移动到 x=1（`Circumscribe`，0.8s）

7. 右侧第2行出现：

   ```
   x=1（跳跃）：
   左极限：lim⁻ f(x) = 0  玫红色
   右极限：lim⁺ f(x) = 1  薄荷色
   极限：不存在（0 ≠ 1）  红色
   ```

   - 玫红探针从左趋近 → 0，薄荷探针从右趋近 → 1（同时动画，1.5s）
   - 两色点尝试汇聚，失败（红色×，0.5s）

8. `Wait(0.8)`

**x=2 处（7秒）：**

9. 黄色圆圈移动到 x=2（`Circumscribe`，0.8s）

10. 右侧第3行：

    ```
    x=2（空洞）：
    左极限：lim⁻ f(x) = 1
    右极限：lim⁺ f(x) = 1
    极限：1  ✓（虽然 f(2)=2）
    ```

    - 两探针趋近 → 都是1，融合为黄色点（1.0s）
    - 函数值 f(2)=2 处的点（白色实心点）闪烁一次，提示"极限≠函数值"

11. `Wait(0.8)`

**x=3 处（7秒）：**

12. 黄色圆圈移动到 x=3 — 0.5s

13. 右侧第4行（快速，`Write`，1.0s）：

    ```
    x=3（连续）：
    lim f(x) = f(3) = 2  ✓
    ```

    - 左右探针汇聚，黄色融合点与函数值重合 — 0.8s

14. `Wait(0.8)`

**x=4 处（7秒）：**

15. 黄色圆圈移动到 x=4 — 0.5s

16. 右侧第5行：

    ```
    x=4（右端点）：
    左极限：lim⁻ f(x) = 1  ✓
    右极限：不存在（函数在右侧无定义）
    极限：1（端点处用单侧极限）
    ```

    - 玫红探针从左趋近 → 1（1.0s）

17. `Wait(0.8)`

**总结停顿（7秒）：**

18. 左侧图中，所有已分析的点各自用对应颜色标注（`LaggedStart`，1.0s）

19. 右侧5行分析表全部可见，`Wait(1.5)`

20. 屏幕下方出现总结（黄色，`Write`，1.0s）：

    > **读一张图，分析五个点——单侧极限让分析变得完整。**

21. **`Wait(2.0)`**

**设计意图：**
这个综合例题是本节的"技能检验"场景——观众跟着逐点分析，建立"我能做这件事"的自信感。黄色圆圈在不同点间的移动，形成视觉上的"逐点扫描"节奏，让观众感受到这是一个系统性的分析方法，而不是每次从头思考。

### 场景 5：半圆函数——端点处的极限（30 秒）

**情绪目标：** "在定义域边界上，单侧极限就是函数的全部"

**布局：** 左侧：半圆图像；右侧：端点极限记号

**视觉元素：**
- 函数 $f(x) = \sqrt{4-x^2}$，图像为上半圆（定义域 $[-2, 2]$）
- 在 x=-2 处：只有右极限（薄荷色）
- 在 x=2 处：只有左极限（玫红色）
- 在 x∈(-2,2) 内：双侧极限均存在

**动画序列：**

1. 上半圆绘制（`Create`，1.5s），两端点（实心点）标注

2. 薄荷色探针从 x=-1 趋近 x=-2（右极限演示）— 1.0s
   - 函数值 → 0
   - 右侧弹出：$\lim_{x\to -2^+}\sqrt{4-x^2} = 0$（薄荷色）

3. 玫红色探针从 x=1 趋近 x=2（左极限演示）— 1.0s
   - 函数值 → 0
   - 右侧弹出：$\lim_{x\to 2^-}\sqrt{4-x^2} = 0$（玫红色）

4. `Wait(1.5)`

5. 注释框出现（白色边框，`Write`，1.5s）：

   ```
   在定义域端点处：
   · 只有"向内"的那侧极限有意义
   · 单侧极限 = 该端点处的极限
   ```

6. **`Wait(2.0)`**

**设计意图：**
这个场景很短，目的是补充"端点处极限"这个边界情形。上半圆是最直观的例子——观众能一眼看出"在-2处只能从右侧趋近"，不需要额外解释。

### 场景 6：★ 核心定理——$\lim_{\theta\to 0}\frac{\sin\theta}{\theta}=1$ 的几何证明（75 秒）

**情绪目标：** 本节情绪最高点——用面积比较来"夹住"一个三角函数的极限，几何的优雅令人惊叹

**布局：** 全屏，以单位圆几何图为主视觉

**视觉元素：**
- 单位圆（白色圆弧，半径1）
- 圆心 O，点 A(1,0)，点 P（圆上，角度 θ）
- 三角形 OAP（蓝色填充）
- 扇形 OAP（黄色填充，实际是扇形区域）
- 三角形 OAT（绿色填充，T 为过 A 的切线与 OP 延长线的交点）
- 面积公式和不等式

**动画序列：**

**建立几何结构（20秒）：**

1. 坐标系出现，绘制单位圆（白色，`Create`，1.5s）

2. 标记点 O（圆心）、A(1,0)（右端点）— 各 0.3s

3. 用 `ValueTracker` 控制角度 θ = 0.7 rad，绘制点 P 在圆上（暖橙色点）— 0.5s

4. 从 P 向 x 轴作垂线，垂足为 Q — 0.5s
   - PQ 的长度 = sin θ（标注，绿色）
   - OQ 的长度 = cos θ（标注，蓝色）

5. 过 A 作竖直切线，延长 OP 与切线交于 T — 0.5s
   - AT 的长度 = tan θ（标注，白色）

6. `Wait(1.0)` — 让观众看清几何结构

**三个面积出场（20秒）：**

7. 填充三角形 OAP（蓝色半透明，`FadeIn`，0.8s）
   - 旁边标注：**面积 = ½ sin θ**（蓝色）

8. 填充扇形 OAP（黄色半透明，`FadeIn`，0.8s）
   - 旁边标注：**面积 = θ/2**（黄色）
   - 小字：*"（θ 以弧度计）"*

9. 填充三角形 OAT（绿色半透明，`FadeIn`，0.8s）
   - 旁边标注：**面积 = ½ tan θ**（绿色）

10. `Wait(1.0)`

**面积不等式（15秒）：**

11. 屏幕右侧出现不等式（逐步 `Write`，每步 0.8s）：

    $$\text{面积}\triangle OAP < \text{面积扇形} OAP < \text{面积}\triangle OAT$$

12. 代入面积公式（每步对应颜色高亮）：

    $$\frac{1}{2}\sin\theta < \frac{\theta}{2} < \frac{1}{2}\tan\theta$$

13. 全除以 $\frac{1}{2}\sin\theta$（动画：三项同时变换，1.5s）：

    $$1 < \frac{\theta}{\sin\theta} < \frac{1}{\cos\theta}$$

14. 取倒数，不等号方向翻转（红色箭头标注翻转，0.8s）：

    $$\cos\theta < \frac{\sin\theta}{\theta} < 1$$

15. `Wait(1.5)`

**夹逼定理应用（10秒）：**

16. 屏幕出现夹逼定理的标志性画面（三色曲线收拢，从场景9 of 2.2节的缩略版）— 0.5s

17. 注释（白色，`Write`，1.0s）：

    > *"当 θ → 0 时：cos θ → 1，上界 1 保持不变"*

18. 两条"曲线"同时趋近 1（蓝色 cos θ 曲线上升、上界 1 保持），中间的 sin θ/θ 被夹住 — 1.5s

19. **结论出现**（黄色，大字，`Write`，1.0s）：

    $$\lim_{\theta \to 0} \frac{\sin\theta}{\theta} = 1$$

20. **`Flash`（黄色光晕，覆盖全屏，0.5s）** — 本节情绪最高点

21. **`Wait(3.0)`** — 全片最长停顿，让这个结论停留

**几何图动态验证（10秒）：**

22. 让 θ 从 0.7 缩小到 0.1（`ValueTracker`，1.5s）：
    - 三个区域同时收缩，越来越小但形状相似
    - 右侧 sin θ/θ 的数值实时显示（`DecimalNumber`），从 ~0.98 趋近 1.000

23. **`Wait(2.0)`**

**设计意图：**
这个证明是本节乃至整个第2章的视觉巅峰——不是因为计算复杂，而是因为**面积的大小关系直接给出了函数值的夹逼**，几何和分析在这里完美相遇。

三个面积用三种颜色（蓝、黄、绿）填充，不仅美观，而且让面积不等式的每一项都有直觉来源。`Flash` 效果在结论出现时释放全片积累的情绪，然后用3秒停顿让观众消化。

动态θ收缩的最后演示，是几何证明的视觉确认——观众亲眼看到三个区域同时趋近同一点。

### 场景 7：sin θ/θ 的应用——系列推论（45 秒）

**情绪目标：** "一个极限，解锁一个系列"——工具的威力在于它的可复用性

**布局：** 左侧：核心公式框；右侧：推论展开

**核心内容：**

- **(a)** $\lim_{y\to 0}\frac{\cos y - 1}{y} = 0$
- **(b)** $\lim_{x\to 0}\frac{\sin 2x}{5x} = \frac{2}{5}$
- **(c)** $\lim_{t\to 0}\frac{\tan t \sec 2t}{3t} = \frac{1}{3}$
- **(d)** $\lim_{\theta\to 0}\frac{\sin A\theta}{\sin B\theta} = \frac{A}{B}$

**动画序列：**

**引入框架（5秒）：**

1. 左侧出现核心公式框（黄色边框）：

   $$\lim_{\theta \to 0} \frac{\sin\theta}{\theta} = 1$$

   - 标注：**"这是接下来所有计算的锚点"**（白色小字）

2. `Wait(0.8)`

**推论(a)：cos y 的极限（12秒）**

3. 右侧出现标题 **(a)**（`Write`，0.5s）

4. 展示推导（逐步，每步 0.8s）：

   $$\frac{\cos y - 1}{y} = \frac{-2\sin^2(y/2)}{y}$$

   - 利用半角公式，白色注释：*"半角公式"*

   $$= -\frac{\sin(y/2)}{y/2} \cdot \sin(y/2)$$

   - 识别 sin θ/θ 结构（黄色框出 `sin(y/2)/(y/2)`）

   $$\xrightarrow{y\to 0} -(1)(0) = 0$$

5. 结论（黄色）：$\lim_{y\to 0}\frac{\cos y-1}{y}=0$ — 0.5s

6. `Wait(0.8)`

**推论(b)：变系数情形（10秒）**

7. 右侧出现 **(b)**（`Write`，0.5s）

8. 关键技巧展示（`Write`，1.5s）：

   $$\frac{\sin 2x}{5x} = \frac{2}{5}\cdot\frac{\sin 2x}{2x}$$

   - 注释（白色）：*"乘以 2/5÷2/5，让分母变成 sin 的自变量"*
   - 黄色框出 `sin 2x/2x`

9. 结论：$= \frac{2}{5}(1) = \frac{2}{5}$ — 0.5s

10. `Wait(0.8)`

**推论(d)：一般公式（10秒）**

11. 右侧出现 **(d)**（`Write`，0.5s）

12. 展示分子分母同乘策略（`Write`，1.5s）：

    $$\frac{\sin A\theta}{\sin B\theta} = \frac{\sin A\theta}{A\theta}\cdot\frac{B\theta}{\sin B\theta}\cdot\frac{A}{B}$$

    - 两个 sin θ/θ 结构各自用黄色框出

13. 结论（黄色）：$\to (1)(1)\cdot\frac{A}{B} = \frac{A}{B}$ — 0.5s

14. `Wait(1.0)`

**总结（8秒）：**

15. 右侧出现总结框（白色边框，`Write`，1.5s）：

    ```
    核心技巧：
    把极限改写为  sin(□)/□  的形式，
    然后令 □ → 0，即可用核心公式。
    ```

16. **`Wait(2.0)`**

**设计意图：**
推论(a)(b)(d)各展示一种变换技巧（半角公式、系数调整、分子分母同乘）。每次都黄色框出"被识别的 sin θ/θ 结构"，让观众看到"原来它藏在这里"。这种"识别→应用"的重复，建立了一种工具使用的肌肉记忆。

## 五、核心视觉语言：颜色融合机制

本节引入的"玫红+薄荷→黄色"颜色融合机制是全节最重要的视觉发明，以下是完整规范：

### 融合规则

| 情形 | 视觉表现 | 含义 |
|:--|:--|:--|
| 左极限存在（值为L） | 玫红色悬浮点 |  $\lim_{x\to c^-} f(x) = L$ |
| 右极限存在（值为L） | 薄荷色悬浮点 | $\lim_{x\to c^+} f(x) = L$ |
| 两侧相等（都是L） | 两点向中心靠拢，融合为黄色大点 + Flash | 双侧极限存在 = L |
| 两侧不等 | 两点各停原处，中间出现红色 × | 双侧极限不存在 |
| 只有一侧有定义 | 只有对应颜色的悬浮点，另一侧无点 | 端点处的极限 |

### 在后续节次中的延续

这个融合机制在2.5节（连续性）将继续使用：

- 若 $\lim_{x\to c}f(x)$ 存在（黄色点）且等于 $f(c)$（白色实心点），两点叠合 → 连续
